## 🗓️ 定期维护清单

> AI 平台 DOM 改版是这类扩展的「最大杀手」,下面的节奏主要围绕「早发现、早修复」。

### 每周(约 5 分钟)

- [ ] 登录 [Chrome Web Store 后台](https://chrome.google.com/webstore/devconsole),看一眼:
  - 本周新增安装 / 卸载量
  - 有无新评论或差评
  - 有无审核警告或违规通知邮件
- [ ] 如有差评 → 截图原文转给我分析修复

### 每月(约 15 分钟)

- [ ] **三平台实跑自测**:分别打开 ChatGPT / Gemini / Claude,各发 2-3 条提问,确认:
  - 右侧目录在 ≥2 条提问时正常出现
  - 点击锚点能跳转
  - 收藏星标能点亮,且刷新后保留
  - Markdown 导出 / 图片导出正常
  - 中英文界面跟随浏览器语言
  - 导出弹窗 / 预览不会被固定目录遮挡
- [ ] 关注 Chrome / Edge 是否有破坏性更新(Chromium 开发者公告)

### 每季度(约 1 小时)

- [ ] **代码 review**:告诉我「再 review 一次」,我扫一遍 `src/content.js`,产出新的优化清单
- [ ] **依赖检查**:`html2canvas.min.js` 是本地打包,检查是否有安全更新或更轻量的替代
- [ ] **汇总用户反馈**:把这段时间积累的评论 / 建议 / bug 整理一次,排迭代优先级
- [ ] 评估是否发新版本(见下方「版本规划」)

## 🚨 平台改版应急流程

哪天 Claude / Gemini / ChatGPT 更新后目录突然失效,按这个流程:

1. 🙋 你告诉我「X 平台坏了」,最好附截图或描述(目录不出现?锚点错位?点击无效?)
2. 🤖 我用 Playwright 打开该平台,定位 DOM 改了哪个 class / 结构
3. 🤖 更新 `src/content.js` 对应选择器(**优先加多级回退,不硬依赖单一 class**)
4. 🤖 `node --check src/content.js` 自检
5. 🙋 你在 `chrome://extensions/` 刷新扩展,确认修复
6. 🤖 视严重程度决定是否紧急发版(bump version → 打包 → 你上传审核)

> 参考:Claude / Gemini 的用户消息和 AI 回复识别都已在 v1.0.2 加入多级回退。

## 🎯 迭代方向(候选)

按价值 / 成本粗略排序,不一定都做,按用户反馈和兴趣挑选。完整清单见 [TODO.md §二](./TODO.md)。

**高价值 · 低成本**
- 收藏列表单独视图:只看星标提问
- 目录搜索 / 过滤:长会话按关键词筛锚点

**中价值 · 中成本**
- 自定义快捷键:打开/关闭面板、跳到上/下一个锚点
- 暗色/亮色主题切换
- 导出为 HTML / PDF

**新平台候选**
- Grok、DeepSeek、Kimi、豆包等更多 AI 对话平台

## 🚢 版本规划

遵循[语义化版本](https://semver.org/lang/zh-CN/):

| 类型 | 触发场景 | 版本号 |
|------|---------|--------|
| Patch | bug 修复、选择器适配改版 | `1.0.x` |
| Minor | 新功能、向后兼容的改进 | `1.x.0` |
| Major | 架构重构、不兼容变更 | `x.0.0` |

**已规划版本**

- `v1.0.2`:已提交审核,重点修复导出 UI、中英双语、图片导出卡死和平台抗改版。
- `v1.1.0`:首批用户反馈收集后的迭代,候选见上方。
